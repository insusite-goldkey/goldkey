## 🧠 지식 구조

### 1. 페르소나 정의
```json
{
  "name": "황금 감옥에 갇힌 성주",
  "target": "5060 CEO",
  "core_problem": "자산의 90%가 법인 주식과 공장 부지에 묶여 있어, 상속세 납부 현금 부족",
  "metaphor": "웅장한 성은 있으나 성문 밖으로 나갈 현금이라는 다리가 끊겨 있음"
}
```

### 2. 상속세 시스템
- **최고세율**: 50%
- **최대주주 할증**: 20% (지분 50% 이상)
- **누진세율**: 1억 이하 10% ~ 30억 초과 50%

### 3. 비상장주식 평가
- **보충적 평가**: 순손익가치 60% + 순자산가치 40%
- **순손익가치**: (최근 3년 평균 순이익 × 3) / 환원율(10%)
- **순자산가치**: 자산 - 부채

### 4. 은유적 후킹 언어
- **상속세**: "국세청의 청구서이자 마지막 세무조사"
- **경영인정기보험**: "상속세 전용 비상금이자 최후의 보루"
- **유동성 부족**: "현금이라는 다리가 끊긴 성"
- **세금 최적화**: "세금 세탁(합법적 절세)"
- **정관 변경**: "법인 정관이라는 방패 수선"

## 🔧 상속세 시뮬레이션 엔진

### 핵심 기능
1. 비상장주식 가치 평가 (1원 단위)
2. 상속세 정밀 계산
3. 현금 부족액 추정
4. 은유적 보고서 생성

### 사용 예시
```python
from engines.ceo_succession_simulator import CEOSuccessionSimulator

simulator = CEOSuccessionSimulator()

# 유동성 위기 시뮬레이션
result = simulator.simulate_liquidity_crisis(
    annual_profit=5_000_000_000,  # 연 순이익 50억
    net_assets=20_000_000_000,    # 순자산 200억
    ownership=0.80,                # 지분 80%
    liquid_assets=2_000_000_000   # 유동자산 20억
)

print(f"상속세액: {result.tax_amount:,}원")
print(f"현금 부족액: {result.cash_shortage:,}원")
print(f"필요 보험금액: {result.insurance_needed:,}원")
print(f"\n{result.metaphor}")
```

**출력 예시**:
```
상속세액: 20,000,000,000원
현금 부족액: 18,000,000,000원
필요 보험금액: 21,600,000,000원

대표님의 성(Castle)은 230억원의 가치를 지니지만, 
국세청의 청구서 200억원을 낼 현금이 180억원 부족합니다. 
이는 대표님의 평생을 건 마지막 세무조사가 될 것입니다.
```

## 🛡️ 전략적 프레임워크

### STEP 7: 가상 상속세 시뮬레이션
**목적**: 법인 가치 평가 → 유고 시 현금 부족액 1원 단위 추정

**핵심 메시지**:
> "단순히 보험료를 뽑지 않습니다. 내가 현재 대표님의 법인 가치를 평가하여, 유고 시 발생할 **현금 부족액**을 1원 단위까지 추정하여 보고하겠습니다."

**계산 공식**:
```
상속세 = (법인가치 × 지분율 × (1 + 최대주주할증)) × 세율 - 공제액
현금 부족액 = 상속세 - 유동자산
```

### STEP 8: 비용 처리의 명분
**목적**: 법인세 절감 효과 입증

**핵심 메시지**:
> "이 보험료는 비용입니다. 국가에 낼 세금을 깎아서 대표님 가족의 **상속 재원**으로 치환하는 **세금 세탁(합법적 절세)**의 과정입니다."

**법인세 절감 효과**:
```
연 보험료 1억 × 법인세율 22% = 2,200만원 절감
실질 부담액 = 1억 - 2,200만원 = 7,800만원
```

### STEP 9: 정관 변경의 칼날 (이중 엔진 전략)
**목적**: 보험금이 법인에서 유가족에게 합법적으로 전달되는 경로 확보

**핵심 메시지**:
> "준비되지 않은 퇴직금 규정은 독이 됩니다. 내가 제안하는 이 플랜은 보험 가입이 아니라, **법인 정관이라는 방패**를 수선하는 작업입니다."

**이중 엔진**:

#### 엔진 1: 퇴직금 지급 규정
- **메커니즘**: 보험금을 법인이 수령 → 임원 퇴직금으로 지급 → 유가족에게 전달
- **법적 근거**: 상법 제388조, 근로기준법 제34조

#### 엔진 2: 자기주식 취득
- **메커니즘**: 보험금으로 자기주식 취득 → 유가족 지분율 유지 → 경영권 방어
- **법적 근거**: 상법 제341조
