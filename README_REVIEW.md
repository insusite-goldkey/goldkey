## 🔐 본인인증 기능 설명 (Beta Simulation Mode)

### 현재 구현 상태

본 앱은 **베타 테스트 기간** 중이며, 본인인증 기능은 **시뮬레이션 모드**로 작동합니다.

#### 1. 시뮬레이션 모드란?

- 실제 PASS 또는 카카오 본인인증 API를 호출하지 않습니다
- 사용자가 입력한 이름과 연락처를 앱 내부 데이터베이스와 대조하여 인증합니다
- 베타 테스트 기간 동안 사용자 편의성을 위해 임시로 적용된 방식입니다

#### 2. 시뮬레이션 프로세스

```
[사용자] 비밀번호 재설정 요청
    ↓
[앱] "베타 테스트용 시뮬레이션 모드입니다" 안내 표시
    ↓
[사용자] PASS 또는 카카오 인증 버튼 클릭
    ↓
[앱] 이름 + 연락처 입력 필드 표시
    ↓
[사용자] 가입 시 사용한 정보 입력
    ↓
[앱] 입력된 연락처를 SHA-256 해시로 변환
    ↓
[앱] GCS 마스터 명부와 해시값 대조
    ↓
[앱] 일치 시 → 새 비밀번호 설정 화면 표시
    ↓
[사용자] 새 비밀번호 입력 및 저장
```

#### 3. 보안 조치

- **연락처 해시 대조**: 평문 연락처는 SHA-256 해시로 변환 후 저장된 해시값과 비교
- **이름 암호화**: 이름은 AES-256으로 암호화되어 저장되며, 인증 시 복호화하여 대조
- **서버 로그 보호**: 평문 연락처는 절대 서버 로그에 기록되지 않음

#### 4. 정식 버전 계획

베타 테스트 종료 후 다음과 같이 업그레이드될 예정입니다:

- **PASS 본인인증 API 연동**: 한국정보인증(주) PASS 공식 API 사용
- **카카오 본인인증 API 연동**: 카카오 공식 본인인증 서비스 사용
- **실명 확인**: 본인인증 기관에서 제공하는 실명 정보로 검증

## 🛡️ 개인정보 보호

### 수집하는 정보

- **필수**: 이름, 연락처, 비밀번호
- **선택**: 기기 핑거프린트 (중복 가입 방지용)

### 암호화 방식

| 항목 | 암호화 방식 | 복호화 가능 여부 |
|------|------------|-----------------|
| 연락처 (로그인 ID) | SHA-256 단방향 해시 | ❌ 불가능 |
| 비밀번호 | SHA-256 단방향 해시 | ❌ 불가능 |
| 이름 | AES-256 양방향 암호화 | ✅ 관리자만 가능 |

### 저장 위치

- **Supabase**: 회원 기본 정보 (암호화된 상태)
- **Google Cloud Storage**: 마스터 명부 백업 (이중 암호화)
